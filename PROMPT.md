# vacation_schedule

# 📋 Архивный промпт: HTML-страница "График отпусков и дежурств сотрудников"

**Роль:** Ты — опытный frontend-разработчик. Создай полноценную интерактивную HTML-страницу (один файл, единственная внешняя зависимость — библиотека SheetJS через CDN) для визуализации графика отпусков и дежурств сотрудников в виде диаграммы Ганта с поддержкой фильтрации по отделам.

## 🎯 Функциональные требования

### 1. Диаграмма Ганта
- По вертикали — строки сотрудников, по горизонтали — календарная шкала.
- Отпуска отображаются цветными полосами с указанием количества дней.
- Дежурства отображаются **поверх** отпусков полупрозрачным чёрным цветом с белой границей.
- Отпуск больше по вертикали (42px), чем дежурство (22px).

### 2. Пересечения отпусков
- При наложении отпусков **разных** сотрудников — отображается красная пунктирная зона **ПОД отпусками** (z-index: 1).
- Логика разделяет пересечения по составу: если A+B пересекаются, а потом B+C — это два разных пересечения.
- Над каждой зоной пересечения — метка **"⚠"** (без количества человек) с тултипом при наведении:
  ```
  Пересечение отпусков (N чел.):
  Сотрудник_1, Сотрудник_2, ...
  ```
- Метка "⚠" имеет `pointer-events: auto` и `z-index: 12`, чтобы реагировать на hover даже поверх отпусков.
- Сама зона пересечения имеет `pointer-events: none`, чтобы клики проходили сквозь неё к отпуску.

### 3. Случайные цвета
- При каждом запуске каждому сотруднику назначается случайный цвет в формате `hsl(hue, saturation%, lightness%)` (hue 0–360, saturation 55–75%, lightness 45–55%).
- **Не сохранять** в `localStorage` — при каждой перезагрузке новые цвета.
- Кнопка "🎲 Цвета" для ручной перегенерации (синего цвета, как кнопки "Дни"/"Недели"/"Месяцы").

### 4. Drag-and-drop сотрудников
- Строки с именами можно перетаскивать вверх/вниз.
- Порядок сохраняется в `localStorage` (ключ `employeeOrder`) — работает только для данных из `rawData`.
- **При загрузке из Excel** порядок сбрасывается (`localStorage.removeItem('employeeOrder')`).
- Визуальная обратная связь: курсор `grab/grabbing`, подсветка, индикатор `↕` справа, ручка `⋮` слева.
- **Важно:** `getFilteredEmployees()` возвращает сотрудников в порядке, сохранённом в `employees`, а не в порядке появления в данных.
- `handleDrop()` работает с основным массивом `employees`, а не с отфильтрованным — это обеспечивает корректную работу drag-and-drop при активной фильтрации по отделам.

### 5. Sticky-элементы
- **Sticky шапка календаря** (месяцы + дни/недели): при вертикальной прокрутке остаётся вверху контейнера.
- **Sticky колонка "Сотрудник"**: при горизонтальной прокрутке остаётся слева.
- Угол "Сотрудник" — sticky по обеим осям (z-index: 11, выше шапки с z-index: 10).
- **Обязательные условия:** у `.chart-wrapper` должен быть `overflow-y: auto` и `max-height`.
- Вертикальный скроллбар тонкий (6px), горизонтальный нормальный (10px).

### 6. Переключение масштаба
- Три кнопки: **Дни** (20px/день), **Недели** (~40px/неделя), **Месяцы** (~120px/месяц).
- **ВАЖНО:** ширина ячеек в нижней строке шапки должна **синхронизироваться** с шириной месяцев в верхней строке:
  - Режим 'day': каждая ячейка = `cellSize` px
  - Режим 'week': каждая ячейка = `7 * cellSize` px (последняя может быть короче)
  - Режим 'month': каждая ячейка = `(endDay - startDay + 1) * cellSize` px
- Использовать `minWidth` и `maxWidth` для жёсткой фиксации ширины.

### 7. Навигация по графику
- Четыре кнопки в порядке: `start` → `left` → `right` → `end`
- Символы: **◀◀** (В начало), **◀** (Назад), **▶** (Вперёд), **▶▶** (В конец)
- У каждой кнопки есть `title` для подсказок при наведении.
- Плавная прокрутка (`behavior: 'smooth'`) на 80% ширины видимой области.
- Кнопки навигации в **сером** стиле (`.nav-button`), отдельно от синих кнопок управления.

### 8. Линия "Сегодня"
- Красная вертикальная линия текущей даты с подписью "Сегодня" сверху.
- Кнопка "👁️ Сегодня" для переключения видимости.
- При нажатии кнопки график **центрируется** на сегодняшнем дне.

### 9. Центрирование на сегодняшнем дне
- Происходит при нажатии кнопок: "Сегодня", "Дни", "Недели", "Месяцы".
- Использовать `setTimeout(() => scrollToToday(), 50)` после `render()`, чтобы DOM успел обновиться.
- Защита от выхода за границы диапазона и нулевых размеров wrapper.
- `scrollToToday()` использует отфильтрованные данные для определения диапазона.

### 10. Тултипы
- При наведении на полосу отпуска/дежурства — всплывающая подсказка с именем сотрудника, типом (Отпуск/Дежурство), датами начала/конца, количеством дней и **отделом** (если указан).
- Тултип следует за курсором, тёмный фон, белый текст.

### 11. Легенда
- Под графиком — список сотрудников с цветовыми квадратами.
- Отдельный пункт "Пересечение отпусков" с красным пунктиром.
- Отдельный пункт "Дежурство" с чёрным цветом.
- **Размер шрифта легенды: 10px.**

### 12. Статистика
- Жёлтый блок: всего отпусков, дежурств, сотрудников, период, количество пересечений.
- Предупреждение, если пересечения есть; подтверждение, если нет.
- Статистика рассчитывается на основе отфильтрованных данных.

### 13. Блок фич
- После статистики — синий блок со списком всех возможностей (grid 2 колонки).

### 14. Загрузка данных из Excel
- Подключить **SheetJS** через CDN: `https://cdnjs.cloudflare.com/ajax/libs/xlsx/0.18.5/xlsx.full.min.js`
- Зелёная кнопка "📥 Загрузить Excel" с `<input type="file">` внутри обёртки.
- **ВАЖНО:** в `XLSX.read` **НЕ использовать** `cellDates: true` — это вызывает проблемы с часовыми поясами.
- Ищет листы "отпуска" и "дежурства" (регистронезависимо, также допускает `vacations` и `duties`).
- Парсит данные: Столбец A — имя, Столбец B — дата, Столбец C — количество дней, **Столбец D — отдел (необязательно)**.
- Пропускает первую строку (заголовки), игнорирует пустые строки.
- После загрузки: обновляет `rawData`/`rawDuty`, сбрасывает порядок сотрудников, перегенерирует цвета, **перерисовывает блок отделов** (`renderDepartmentsFilter()`), перерисовывает график.
- Показывает сообщение об успехе/ошибке (авто-скрытие через 5 секунд).

### 15. ⚠️ Работа с датами и часовыми поясами (КРИТИЧНО!)
**Принцип: везде локальное время, кроме расчёта разницы в днях.**

- **`parseData`:** создаёт даты через `new Date(y, m-1, d)` — локальное время, полночь.
- **`daysBetween`:** использует `Date.UTC` для точного расчёта разницы в днях (защита от DST):
  ```javascript
  function daysBetween(d1, d2) {
    const utc1 = Date.UTC(d1.getFullYear(), d1.getMonth(), d1.getDate());
    const utc2 = Date.UTC(d2.getFullYear(), d2.getMonth(), d2.getDate());
    return Math.round((utc2 - utc1) / (1000 * 60 * 60 * 24));
  }
  ```
- **`formatDate`:** использует **локальные методы** (`getDate()`, `getMonth()`, `getFullYear()`), НЕ UTC!
- **Excel-парсинг:** конвертация дат через локальные методы:
  ```javascript
  if (dateValue instanceof Date) {
    const d = String(dateValue.getDate()).padStart(2, '0');
    const m = String(dateValue.getMonth() + 1).padStart(2, '0');
    const y = dateValue.getFullYear();
    dateStr = `${d}.${m}.${y}`;
  } else if (typeof dateValue === 'number') {
    const excelDate = new Date(Math.round((dateValue - 25569) * 86400 * 1000));
    const d = String(excelDate.getDate()).padStart(2, '0');
    const m = String(excelDate.getMonth() + 1).padStart(2, '0');
    const y = excelDate.getFullYear();
    dateStr = `${d}.${m}.${y}`;
  }
  ```
- **Линия "Сегодня":** `const today = new Date(); today.setHours(0, 0, 0, 0);`
- **`minDate`/`maxDate`:** создаются через локальные методы, без `setDate()`/`setMonth()` после создания.

### 16. Фильтрация по отделам
- Блок расположен **после легенды**, перед статистикой.
- Содержит:
  - Заголовок "🏢 Фильтр по отделам:"
  - Кнопки "Выбрать все" / "Снять все"
  - Список чекбоксов с названиями отделов и количеством **уникальных сотрудников** в каждом
- Пустой отдел отображается как "(без отдела)" и сортируется в конец списка.
- При изменении фильтра график автоматически перерисовывается.
- Пересечения считаются только для отфильтрованных отпусков.
- Диапазон дат пересчитывается на основе отфильтрованных данных.
- Функция `getDepartmentCounts()` использует `Set` для подсчёта уникальных сотрудников (не количество записей).
- Функция `getFilteredEmployees()` возвращает сотрудников в порядке, сохранённом в `employees`.

### 17. Подсказка (tooltip) для кнопки загрузки Excel
- При наведении на кнопку "📥 Загрузить Excel" появляется подробная подсказка с описанием работы функции и ожидаемой структуры Excel-файла.
- Подсказка следует за курсором, но **никогда не выходит за пределы экрана**:
  - Если не помещается справа — показывается слева от курсора.
  - Если не помещается снизу — показывается выше курсора.
  - Минимальный отступ от краёв экрана — 10–15px.
- Проверка всех 4 сторон: `left`, `right`, `top`, `bottom`.

### 18. Активные состояния кнопок
- Кнопки **"👁️ Пересечения"**, **"👁️ Сегодня"**, **"👁️ Дежурства"** — подсвечены тёмно-синим (`#2980b9`), когда соответствующий элемент виден на графике.
- Кнопки **"Дни"**, **"Недели"**, **"Месяцы"** — подсвечена только одна (текущий режим отображения).
- Стиль активного состояния: `background: #2980b9` + `box-shadow: inset 0 2px 4px rgba(0,0,0,0.15)` для эффекта "вдавленности".
- Функция `updateButtonStates()` вызывается при каждом переключении и при инициализации.

- **Метка "⚠":** белый фон, красная рамка, `pointer-events: auto`, эффект увеличения при hover.
- **Сообщения:** зелёный/красный/синий блок с левой границей, авто-скрытие через 5 секунд.
- **Блок фильтрации по отделам:** синий фон (`#f0f7ff`), левая граница 4px синего цвета.

## ⚙️ Технические требования

- Один HTML-файл, всё встроено (CSS в `<style>`, JS в `<script>`).
- Единственная внешняя зависимость — SheetJS через CDN.
- Vanilla JavaScript, без ES6-модулей.
- Расчёт диапазона: от первого дня месяца минимальной даты до последнего дня месяца максимальной даты.
- Grid-разметка для графика: `grid-template-columns: 180px <ширина_таймлайна>px`.
- Переменные `vacations`, `duties`, `employees`, `minDate`, `maxDate` — изменяемые (`let`).
- Переменная `selectedDepartments` — `Set` с выбранными отделами для фильтрации.

## 🧩 Структура кода

1. **CSS** — все стили в `<style>`, включая кастомные скроллбары, стили сообщений, тултипов, блока фильтрации по отделам, активных состояний кнопок.
2. **HTML** — заголовок, блок `.controls` (с кнопками в правильном порядке и `id` для toggle-кнопок), блок `#message`, `.chart-wrapper`, `.legend`, `.departments-filter`, `.stats`, `.features`, два `.tooltip` (основной и для Excel).
3. **JS:**
   - Константы `rawData` и `rawDuty` с 4 столбцами (включая отдел).
   - Функция `parseData()` — парсинг строк в массив объектов `{name, start, end, days, department}`.
   - Переменные состояния: `showOverlap`, `showTodayLine`, `showDuty`, `viewMode`, `draggedEmployee`, `dragSrcIndex`, `colors`, `employees`, `vacations`, `duties`, `minDate`, `maxDate`, `selectedDepartments`.
   - Функции цветов: `generateRandomColor()`, `assignRandomColors()`, `loadColors()`, `reshuffleColors()`.
   - Функции отделов: `getDepartments()`, `getDepartmentCounts()` (с `Set` для уникальных сотрудников), `initDepartmentsFilter()`, `toggleDepartment()`, `selectAllDepartments()`, `deselectAllDepartments()`, `getFilteredVacations()`, `getFilteredDuties()`, `getFilteredEmployees()` (возвращает в порядке `employees`), `renderDepartmentsFilter()`.
   - Функция `countOverlaps()` — корректный подсчёт пересечений с разделением по составу.
   - Функция `getCellSize()` — размер ячейки в зависимости от масштаба.
   - Функция `handleFileUpload(event)` — загрузка и парсинг Excel через SheetJS (без `cellDates: true`, с чтением 4-го столбца "Отдел"), вызов `renderDepartmentsFilter()` после загрузки.
   - Функция `showMessage(text, type)` — показ сообщений.
   - Функция `render()` — полная перерисовка графика с использованием отфильтрованных данных.
   - Обработчики drag-and-drop: `handleDragStart/End/Over/Enter/Leave/Drop` (работают с основным массивом `employees`).
   - Обработчики tooltip для кнопки Excel: `mouseenter`, `mousemove`, `mouseleave` с умным позиционированием.
   - Функции управления: `toggleOverlap()`, `toggleToday()`, `toggleDuty()`, `changeView()`, `scrollChart()`.
   - Функция `scrollToToday()` — центрирование на сегодняшнем дне (использует отфильтрованные данные).
   - Функция `updateButtonStates()` — обновление активных состояний кнопок.
   - Функции тултипа отпусков: `showTooltip()` (показывает отдел), `hideTooltip()`.
   - Сохранение/загрузка порядка: `saveEmployeeOrder()`, `loadEmployeeOrder()`.
   - Инициализация в конце: `loadEmployeeOrder(); loadColors(); initDepartmentsFilter(); renderDepartmentsFilter(); render(); updateButtonStates();`.
